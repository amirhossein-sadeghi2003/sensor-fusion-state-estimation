# Project Report: Sensor Fusion State Estimation

## 1. Project Goal

This project demonstrates 2D sensor fusion and state estimation for an autonomous motion tracking scenario.

The goal is to estimate the position and velocity of a moving platform using:

- a simulated 2D motion model
- noisy GPS-like position measurements
- noisy IMU-like acceleration measurements
- a Kalman Filter estimator
- RMSE-based evaluation
- GPS dropout and noise-level experiments

The project is inspired by robotics, autonomous systems, and aerospace-style navigation problems, where a system must estimate its state from noisy and incomplete sensor measurements.

---

## 2. Motivation

Many real-world intelligent physical systems cannot rely on perfect sensor data.

Examples include:

- mobile robots
- drones
- autonomous vehicles
- aerospace navigation systems
- embedded monitoring platforms
- cyber-physical systems

In these systems, sensors may be noisy, delayed, low-rate, or temporarily unavailable.

A state estimator combines a motion model with sensor measurements to produce a better estimate of the system state than raw measurements alone.

This project explores that idea in a controlled simulation environment where the true trajectory is known, making quantitative evaluation possible.

---

## 3. System Overview

The simulated platform moves in a 2D plane.

The project workflow is:

```text
simulate true 2D motion
→ generate GPS-like noisy position measurements
→ generate IMU-like noisy acceleration measurements
→ simulate GPS dropout
→ run Kalman Filter prediction and correction
→ estimate position and velocity
→ compare raw GPS and Kalman estimates
→ evaluate RMSE
→ run GPS noise-level comparison
```

The state vector is:

```text
x = [position_x, position_y, velocity_x, velocity_y]
```

The Kalman Filter estimates this state over time.

---

## 4. Motion Simulation

The true motion is generated using a scripted 2D acceleration profile.

The profile includes:

- acceleration phase
- turning-like motion
- deceleration phase
- direction changes
- final steady segment

This creates a trajectory that is simple enough to understand but still non-trivial for state estimation.

Simulation settings:

| Parameter | Value |
|---|---:|
| Duration | 60 s |
| Time step | 0.1 s |
| Number of samples | 601 |

Generated true states include:

- `true_x_m`
- `true_y_m`
- `true_vx_mps`
- `true_vy_mps`
- `true_ax_mps2`
- `true_ay_mps2`

## 5. Sensor Models

### GPS-like Position Sensor

The GPS-like sensor provides noisy position measurements:

```text
gps_x_m
gps_y_m
```

The GPS-like sensor is lower-rate than the simulation timestep.

It is available every 5 simulation steps and is unavailable during a dropout interval.

Default GPS noise standard deviation:

```text
3.0 m
```

GPS dropout interval:

```text
25 s to 35 s
```

This models temporary loss of position measurements.

---

### IMU-like Acceleration Sensor

The IMU-like sensor provides acceleration measurements:

```text
imu_ax_mps2
imu_ay_mps2
```

These measurements are generated by adding Gaussian noise to the true acceleration values.

Default IMU acceleration noise standard deviation:

```text
0.15 m/s²
```

The IMU-like acceleration measurements are used as control inputs in the Kalman Filter prediction step.

---

## 6. Kalman Filter Design

The Kalman Filter uses a constant-velocity motion model with acceleration input.

Prediction model:

```text
x_k = F x_{k-1} + B u_k
```

where:

```text
u_k = [acceleration_x, acceleration_y]
```

The GPS-like position measurement model is:

```text
z_k = H x_k
```

where:

```text
z_k = [gps_x, gps_y]
```

The filter performs two stages:

1. Prediction using the motion model and IMU-like acceleration input
2. Correction when GPS-like position measurements are available

During GPS dropout, the correction step is skipped and the filter relies only on prediction.

This allows the project to show how state estimation behaves when measurements are temporarily unavailable.

```text

```

The goal is to compare:

- full Kalman trajectory estimate
- Kalman estimate at GPS measurement timestamps

---

### Experiment 2: GPS Dropout Behavior

The simulation includes a GPS dropout interval from 25 s to 35 s.

During this interval:

- GPS-like measurements are unavailable
- the Kalman Filter only performs prediction
- position error increases
- after GPS returns, the filter corrects the estimate

This demonstrates a realistic state-estimation challenge.

---

### Experiment 3: GPS Noise-Level Comparison

The project also evaluates estimation performance under different GPS-like noise levels.

Tested GPS noise standard deviations:

```text
1 m, 2 m, 3 m, 5 m, 8 m, 12 m
```

Each noise level is repeated 5 times with different random seeds.

Outputs:

```text
results/noise_level_comparison.csv
results/noise_level_rmse_comparison.png
```

The goal is to evaluate whether the Kalman Filter consistently improves over raw GPS-like measurements as sensor noise increases.

## 8. Results Summary

The default simulation shows that the Kalman Filter improves position estimation compared with raw GPS-like measurements.

Current RMSE results:

| Method | Position RMSE |
|---|---:|
| Raw GPS-like measurements | 4.51 m |
| Kalman filter estimate over full trajectory | 2.61 m |
| Kalman estimate at GPS timestamps | 1.82 m |

Interpretation:

- raw GPS-like measurements are noisy
- Kalman estimates are smoother and more accurate
- full-trajectory RMSE includes the GPS dropout interval
- error increases during dropout but decreases again when GPS returns

The noise-level experiment also shows that the Kalman Filter reduces RMSE across multiple GPS noise levels.

Example results:

| GPS Noise Std | Raw GPS RMSE | Kalman Full-Trajectory RMSE |
|---:|---:|---:|
| 1 m | 1.38 m | 0.83 m |
| 3 m | 4.13 m | 1.99 m |
| 8 m | 11.01 m | 5.17 m |
| 12 m | 16.51 m | 7.75 m |

This confirms the benefit of combining a motion model with noisy sensor data.

## 10. Engineering Interpretation

This project demonstrates a core idea in intelligent physical systems:

```text
a physical motion model + noisy sensors + estimation algorithm
→ improved state estimate
```

The Kalman Filter is useful because it does not rely only on raw measurements.

Instead, it balances:

- model-based prediction
- measurement-based correction
- uncertainty in process dynamics
- uncertainty in sensor measurements

This makes the project relevant to:

- control systems
- robotics
- autonomous systems
- aerospace-inspired navigation
- embedded sensing
- cyber-physical systems

---

## 11. Limitations

Current limitations:

- the trajectory is simulated, not collected from real hardware
- the motion model is linear
- the filter uses a constant-velocity model with acceleration input
- GPS and IMU sensors are simplified
- IMU bias and drift are not yet modeled
- sensor timing issues are not modeled
- no real embedded implementation is included yet
- the current version uses a standard Kalman Filter, not an Extended Kalman Filter

These limitations are acceptable for a first state-estimation project, but they motivate future extensions.
